```
本日は「フォト婚オールインワンプラットフォーム」というシステム案を提案させていただきます。
Creepy Codeと申します。よろしくお願いします。

私たちが注目したのは、ブライダル、中でも「フォト婚」と呼ばれる写真だけの結婚記念サービスです。

このサービスが解決しようとしているのは、「撮影場所・カメラマン・衣装を別々に探さなければいけない」「夜中に調べても問い合わせができない」といった、ユーザーが長年抱えてきた不満です。

AIプランナーを搭載し、スマホひとつで全部完結できる仕組みを目指します。それでは発表を始めます。
```

## ▶ スライド 3｜セクション01「業界を選んだ理由」

```
最初のセクションです。「なぜブライダル業界、それもフォト婚に着目したのか」をご説明します。

一言でいうと、「市場は拡大しているのに、ユーザーの不満が長年解決されていない領域」だからです。

ITを使えば直接解決できる課題が明確にあり、私たちの技術が活かせると判断しました。

具体的な理由を4点に整理しました。次のスライドでご覧ください。
```

## ▶ スライド 4｜業界を選んだ理由（詳細）

```
理由は4つあります。

1点目、「成長市場」です。婚姻件数は減っていますが、フォト婚の実施率は74.4%と過去最高。市場規模も1,025億円に達しています。

2点目、「課題が明確」です。複数サービスを掛け持ちする手間、夜間に問い合わせできない問題、費用が見えないストレス。どれもITで解決できます。

3点目、「技術適性」です。AIチャット・予約システム・データ管理は、私たちが学んでいる技術そのものです。

4点目、「ターゲットが明確」です。20代後半から30代の共働きカップルに絞ることで、設計がぶれません。
```

## ▶ スライド 5｜セクション02「業界の現状と課題」

```
次のセクションです。「業界が今どうなっているか」と「ユーザーが何に困っているか」を見ていきます。

ここで重要なのは、市場の数字だけでなく、実際に使うユーザーの視点です。

いくら市場が伸びていても、ユーザーが不満を抱えたままでは本質的な解決にはなりません。データとユーザーの声、両方から課題を整理しました。
```

---

## ▶ スライド 6｜業界の現状データ

```
まず市場データをご覧ください。

婚姻件数は2024年に48.5万組。ピーク時の半分以下です。一方でフォト婚の実施率は74.4%と過去最高を記録しており、市場規模は1,025億円に拡大しています。

つまり「結婚式をやる人は減っているが、写真を撮る人は増えている」というのが現状です。

さらに平均費用も年々上昇しており、1組あたり28.4万円。高単価化も進んでいます。この成長セグメントにITで切り込むのが今回の提案です。
```

---

## ▶ スライド 7｜ユーザーが感じる課題

```
では実際にユーザーは何に困っているのでしょうか。大きく4つの課題があります。

1つ目、「情報の分散」。場所・カメラマン・衣装をそれぞれ別のサービスで探す必要があり、手間が大きい。

2つ目、「営業時間の壁」。電話が苦手な20代や、夜勤明けに調べたい30代が多く、夜間は問い合わせできない。

3つ目、「費用の不透明感」。初回見積もりが低くて後から追加費用が増える慣行がある。

4つ目、「日程調整の手間」。3者の空き日程を自分で合わせるのが非常に煩雑。

これら4つを、1つのシステムで同時に解決します。
```

## ▶ スライド 8｜セクション03「システム案の提案」

```
いよいよ核心のセクションです。「どんなシステムを作るのか」をご説明します。

提案するのは、フォト婚に特化したオールインワンプラットフォームです。

最大の特徴は2点。「3つの予約モジュールを1サービスに統合すること」と、「AIプランナーを搭載してサイト内で完結させること」です。

従来は「プランナーに電話して来店して」という流れが当たり前でしたが、そこをAIで代替します。
```

---

## ▶ スライド 9｜システム概要

```
システムは3つのモジュールで構成されます。

「場所」「カメラマン」「衣装」の3つです。

場所は地図検索と空き状況をリアルタイムで確認できます。カメラマンはポートフォリオと口コミを見て選べます。衣装はサイズ・カラーの在庫をオンラインで確認し、セット割引でまとめて予約できます。

そして最大のポイントは、選択するたびに費用の総額がリアルタイムで更新されることです。「最終的にいくらかかるか分からない」という不安を、UIの設計で解消します。
```

---

## ▶ スライド 10｜AIプランナー「ハナ」

```
このシステムの差別化ポイントが、AIプランナー「ハナ」です。

従来はプランナーに電話・来店しないと進まなかった相談が、チャットで24時間対応できるようになります。

「森の中でナチュラルな雰囲気、ドレスはオフホワイト、予算30万」と入力するだけで、AIがプランを自動生成します。

また、費用シミュレーション・スケジュール最適化・ビジュアル診断・準備リマインド・FAQ対応の6つの機能を搭載します。これにより「深夜に調べてそのまま予約完了」という体験が実現できます。
```

---

## ▶ スライド 11｜システムで解決できること

```
先ほどお伝えした4つの課題が、このシステムでどう解決されるかを整理します。

「情報の分散」→ 3モジュール一元予約で、複数サービスの掛け持ちが不要になります。

「営業時間の壁」→ AIが24時間対応するので、深夜でも相談・予約が完了できます。

「費用の不透明感」→ リアルタイム費用シミュレーターで、選ぶたびに総額が更新されます。

「日程調整の手間」→ 3者の空き日程をAIが自動照合し、候補日を即提示します。

課題と解決策が1対1で対応しているのが、このシステムの強みです。
```

```
最後に、このシステムによって期待される効果をまとめます。

チャット開始から予約完了までのCVR（転換率）は15%以上を目標にしています。ペルソナAの平均受注額は22〜28万円、こだわり層のペルソナBは38〜50万円を見込んでいます。

また、納品後のSNSシェア率を40%以上にする設計にすることで、SNS経由の口コミ集客エンジンに育てます。

ユーザー側には「手間ゼロ・安心感・自由度」を、提供側には「業務効率化・集客強化・データ蓄積」をそれぞれ提供できます。
```
